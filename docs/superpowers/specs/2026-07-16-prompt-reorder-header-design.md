# 提示词排序标题栏设计

## 目标

提示词列表因长按进入拖动排序状态后，把排序相关操作集中到标题栏，避免页面底部出现重复且不易触达的取消按钮。

## 交互

- 普通状态保持现状：左侧显示返回按钮，中间显示“提示词”，右侧显示新建按钮。
- 长按任意提示词或分组进入排序状态后：
  - 左侧用文字按钮“取消”替代返回按钮；
  - 中间隐藏“提示词”；
  - 右侧继续显示“完成”；
  - 页面内容区不再显示底部“取消排序”按钮。
- 点击标题栏左侧“取消”时，放弃当前未保存的拖动结果，并恢复普通状态。
- 点击右侧“完成”时，立即显示微信原生居中 Loading，文案为“保存中”，并使用遮罩阻止重复操作；网络提交结束后始终关闭 Loading。
- 保存成功后恢复普通状态；保存冲突或失败时保留现有错误提示和排序状态，允许用户重试或取消。

## 实现边界

- 扩展共享 `page-header` 组件，使调用页面可以隐藏默认返回按钮、隐藏标题，并提供左侧插槽；默认行为不变，避免影响其他页面。
- 提示词页面仅在 `reordering` 为真时启用左侧取消操作和隐藏标题。
- 不修改拖动排序、保存、冲突处理或提示词数据契约。
- 不修改 Android、iOS 或后端仓库；本次仅调整微信小程序界面。

## 验证

- 自动化测试确认普通状态仍声明“提示词”标题和新建按钮。
- 自动化测试确认排序状态隐藏默认返回和标题、在左侧显示“取消”、右侧显示“完成”，且内容区不存在底部取消按钮。
- 自动化测试确认完成排序时显示带遮罩的“保存中”Loading，并在成功和失败结果下关闭 Loading。
- 运行 `npm test` 和 `npm run validate:miniapp`。
- 微信开发者工具中手动验证长按进入排序、取消恢复原顺序、完成保存顺序以及胶囊区域不遮挡。
